Comprehensive Overview of Linux Commands

Linux commands are essential tools for interacting with the Linux operating system. Below is a detailed list of commonly used Linux commands, categorized by their functionality. Each command is described with its purpose to help users understand and utilize them effectively:

1. File Management

ls

Lists the files and directories in the current directory.

cd

Changes the current directory.

pwd

Prints the current working directory.

mkdir

Creates a new directory.

rmdir

Removes an empty directory.

rm

Removes files or directories.

cp

Copies files or directories from one location to another.

mv

Moves or renames files or directories.

touch

Creates an empty file or updates the timestamp of an existing file.

find

Searches for files and directories in a directory hierarchy.

locate

Finds files by name using a prebuilt database.

updatedb

Updates the database used by the locate command.

2. File Viewing and Editing

cat

Concatenates and displays the contents of files.

less

Views file contents one screen at a time.

more

Views file contents one page at a time, similar to less.

nano

A simple text editor for editing files.

vim

A powerful text editor with advanced features.

emacs

A highly customizable text editor with extensive features.

grep

Searches for specific patterns within files.

sed

Stream editor for filtering and transforming text.

awk

Pattern scanning and processing language for text data.

3. File Permissions

chmod

Changes the permissions of files or directories.

chown

Changes the owner of files or directories.

chgrp

Changes the group ownership of files or directories.

4. System Information

uname

Displays system information.

top

Displays a real-time view of system processes and resource usage.

htop

An enhanced version of top with a more user-friendly interface.

df

Displays disk space usage for file systems.

du

Displays disk usage of files and directories.

free

Displays memory usage.

uptime

Shows how long the system has been running.

lscpu

Displays CPU architecture information.

lsblk

Lists information about block devices.

5. Process Management

ps

Displays information about active processes.

top

Displays a real-time view of system processes and resource usage.

htop

An enhanced version of top with a more user-friendly interface.

kill

Sends a signal to terminate a process.

pkill

Kills processes by name.

killall

Kills all processes with a specific name.

bg

Resumes a suspended job in the background.

fg

Brings a background job to the foreground.

jobs

Lists all background and suspended jobs.

6. Networking

ping

Tests connectivity to a network host.

ifconfig

Displays or configures network interfaces (deprecated; use ip instead).

ip

Displays or configures network interfaces and routing (replacement for ifconfig).

netstat

Displays network connections, routing tables, and interface statistics.

ss

Displays socket statistics (replacement for netstat).

curl

Transfers data from or to a server using various protocols.

wget

Downloads files from the web.

traceroute

Displays the route packets take to a network host.

nslookup

Queries DNS to obtain domain name or IP address information.

7. Package Management

apt-get

Manages packages on Debian-based systems (e.g., Ubuntu).

apt

A more user-friendly front-end for apt-get.

yum

Manages packages on RPM-based systems (e.g., CentOS, Fedora).

dnf

Next-generation package manager for RPM-based systems (replaces yum).

rpm

Manages RPM packages on RPM-based systems.

zypper

Manages packages on openSUSE and SUSE Linux Enterprise.

pacman

Manages packages on Arch Linux and derivatives.

8. System Administration

sudo

Executes a command with superuser privileges.

su

Switches to another user account.

passwd

Changes a userâ€™s password.

useradd

Adds a new user to the system.

usermod

Modifies a user account.

userdel

Deletes a user account.

groupadd

Adds a new group.

groupmod

Modifies a group.

groupdel

Deletes a group.

crontab

Schedules periodic tasks.

systemctl

Controls the systemd system and service manager.

service

Manages services on older systems (replaces systemctl).

shutdown

Shuts down or reboots the system.

reboot

Reboots the system.

hostname

Shows or sets the system's hostname.

9. Compression and Archiving

tar

Archives files into a tarball and extracts files from a tarball.

gzip

Compresses files using the gzip compression algorithm.

bzip2

Compresses files using the bzip2 compression algorithm.

xz

Compresses files using the xz compression algorithm.

zip

Creates a zip archive and extracts files from zip archives.

unzip

Extracts files from zip archives.

10. Miscellaneous

echo

Displays a line of text or a variable value.

history

Shows the command history.

alias

Creates shortcuts for longer commands.

man

Displays the manual for a command.

help

Provides help information about shell built-in commands.

which

Shows the path of the executable file for a command.

whereis

Locates the binary, source, and manual page files for a command.

type

Displays the type of a command (e.g., alias, function, builtin, file).

tar

Archives files into a tarball and extracts files from a tarball.

Linux Interview Questions

Basic Questions

Q1: What is Linux?

A1: Linux is an open-source Unix-like operating system kernel that serves as the foundation for various operating systems, collectively known as Linux distributions.

Q2: What is the Linux kernel?

A2: The Linux kernel is the core part of the Linux operating system. It manages hardware resources, provides essential services, and facilitates interactions between software and hardware.

Q3: What is a Linux distribution?

A3: A Linux distribution (or distro) is a version of the Linux operating system that includes the Linux kernel and additional software packages, utilities, and tools. Examples include Ubuntu, CentOS, and Fedora.

Q4: How do you check the current version of Linux?

A4: Use the command uname -r to display the current kernel version, or cat /etc/os-release to get detailed information about the distribution.

Q5: What is the purpose of the ls command?

A5: The ls command lists the files and directories in the current directory.

Q6: How do you create a new directory in Linux?

A6: Use the command mkdir followed by the directory name. For example, mkdir new_directory.

Q7: What does the pwd command do?

A7: The pwd command prints the current working directory.

Q8: How can you delete a file in Linux?

A8: Use the command rm followed by the file name. For example, rm file.txt.

Q9: What is the function of the cp command?

A9: The cp command copies files or directories from one location to another.

Q10: How do you move or rename a file in Linux?

A10: Use the mv command followed by the source and destination paths. For example, mv old_name.txt new_name.txt.

Q11: What is the purpose of the cat command?

A11: The cat command concatenates and displays the content of files.

Q12: How do you display the first few lines of a file?

A12: Use the head command followed by the file name. For example, head file.txt.

Q13: How do you display the last few lines of a file?

A13: Use the tail command followed by the file name. For example, tail file.txt.

Q14: What does the grep command do?

A14: The grep command searches for patterns within files and outputs matching lines.

Q15: How do you find the manual page for a command?

A15: Use the man command followed by the command name. For example, man ls.

Q16: What is the function of the chmod command?

A16: The chmod command changes the permissions of files or directories.

Q17: How do you change the owner of a file?

A17: Use the chown command followed by the new owner and the file name. For example, chown user file.txt.

Q18: How do you check disk space usage?

A18: Use the df command to display disk space usage for file systems.

Q19: What is the purpose of the du command?

A19: The du command estimates and displays the disk space used by files and directories.

Q20: How do you search for files by name?

A20: Use the find command with the -name option. For example, find /path -name filename.txt.

Intermediate Questions

Q21: What does the ps command do?

A21: The ps command displays information about currently running processes.

Q22: How do you kill a process?

A22: Use the kill command followed by the process ID (PID). For example, kill 1234.

Q23: How can you view active network connections?

A23: Use the netstat or ss command to view active network connections.

Q24: What is the crontab used for?

A24: The crontab command manages cron jobs, which are scheduled tasks that run at specified intervals.

Q25: How do you check for open ports on a system?

A25: Use the netstat -tuln or ss -tuln command to list open ports and listening services.

Q26: What does the lsof command do?

A26: The lsof command lists open files and the processes that opened them.

Q27: How can you find out which package a file belongs to?

A27: Use the rpm -qf command on RPM-based distributions or dpkg -S on Debian-based distributions. For example, rpm -qf /path/to/file.

Q28: How do you check system performance?

A28: Use commands like top, htop, or vmstat to monitor system performance and resource usage.

Q29: What is the difference between hard link and soft link?

A29: A hard link is an additional directory entry for a file, sharing the same inode. A soft link (symbolic link) is a separate file that points to another file by its path.

Q30: How do you display the disk usage of a specific directory?

A30: Use the du -sh directory_name command to show the disk usage of a specific directory in a human-readable format.

Advanced Questions

Q31: What is the role of the kernel in Linux?

A31: The kernel manages system resources, facilitates communication between hardware and software, and provides system services.

Q32: How do you check kernel parameters in Linux?

A32: Use the sysctl command to view and modify kernel parameters. For example, sysctl -a lists all parameters.

Q33: How do you configure a network interface in Linux?

A33: Configuration varies by distribution. On Debian-based systems, edit /etc/network/interfaces. On Red Hat-based systems, use nmcli or edit /etc/sysconfig/network-scripts/ifcfg-eth0.

Q34: What is a Linux init system?

A34: The init system is responsible for initializing system services during boot. Common init systems include SysVinit, Upstart, and systemd.

Q35: How do you manage system services with systemd?

A35: Use the systemctl command to manage services. For example, systemctl start service_name starts a service.

Q36: What is the selinux and how is it configured?

A36: SELinux (Security-Enhanced Linux) provides enhanced security through mandatory access controls. Configuration is managed through /etc/selinux/config and commands like setenforce.

Q37: How do you create a swap file in Linux?

A37: Create a swap file with dd if=/dev/zero of=/swapfile bs=1M count=1024, set it up with mkswap /swapfile, and enable it with swapon /swapfile.

Q38: What is the purpose of the iptables command?

A38: iptables is used to configure network packet filtering rules and manage firewall settings in Linux.

Q39: How do you list all installed packages on a Debian-based system?

A39: Use the dpkg -l command to list all installed packages.

Q40: What is a Linux container and how does it differ from a virtual machine?

A40: A Linux container is a lightweight, isolated environment that shares the host OS kernel. It differs from a virtual machine, which includes a full OS and its own kernel.
